# 服务器搭建 chatgpt_web

# 预了解（docker+ninja+clash）

目的: 在自己的服务器上实现一个 web 端 chatgpt，可以不挂 vpn 使用 chatgpt，也可以与自己的好朋友共用。

呈现:

准备：

  1. 一个 chatgpt 账号：用于提供 API key 或者 token 二选一即可，本文选择 token 方式

  2. 一个代理：服务端需要设置代理与 OPAI 服务器通信

优点： 简单、易上手、实用性强、全 docker 操作

# 注意与问题：

  1. 有些时候由于 clash 网络问题，导致 chatgpt_web 端出现反向代理错误，这个时候，需要重启 clash 容器、反向代理容器、chatgpt_web 容器

  2. token 获取：

    • 设置本机全局代理
    • 浏览器无痕模式访问 https://chat.openai.com/auth/login
    • 登录成功之后，在新页面访问 https://chat.openai.com/api/auth/session 就可以得到长串 token，复制过去即可。

    info token 会过期，需要重新操作。
